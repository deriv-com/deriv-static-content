Elevating security measures for your API access

Dear {{ Profile.firstName | default:"trader" }},

At Deriv, we are committed to ensuring the highest level of security for your data and interactions with our platform. As part of our ongoing efforts to implement best security practices, we are introducing a change to the way API Tokens are accessed.

Previously, users could access, view, copy, and delete API Tokens anytime after creation. In our continuous pursuit of enhancing security measures, we have now implemented a more robust approach. Moving forward, users will only be able to view and copy the API Token once immediately after its creation.

This adjustment is designed to reduce the potential risks associated with prolonged access to sensitive information. By limiting the window for viewing and copying API Tokens, we aim to enhance the overall security posture of our platform and, subsequently, safeguard your valuable data.

Key points to note about this change:

Access Window: Users will now have a one-time opportunity to view and copy the API Token immediately after its creation. Subsequent access for these actions will not be available.

Enhanced Security: This change is aligned with industry best practices to mitigate the risk of unauthorized access and misuse of API Tokens, providing an additional layer of protection for your account.

Adapting to Industry Standards: As security threats evolve, so do our security measures. This change reflects our commitment to staying ahead of potential risks and ensuring the highest level of security for your data.

We understand that security is a top priority for you, and we appreciate your understanding and cooperation as we implement these improvements.
